# Общие требования к выполнению задания

* Ноутбук должен быть структурирован: используйте заголовки **1–2 уровней** по разделам задания.
* Под каждой ячейкой, которая должна что-то выводить, должен быть **показан вывод**.
* Код должен сопровождаться **краткими комментариями** (внутри ячейки или текстом перед ней).

Для выполнения работы предлагается воспользоваться сервисом **OpenRouter**, который предоставляет API для взаимодействия с различными LLM и другими моделями.

Из списка доступных моделей можно воспользоваться бесплатными, например:

```
https://openrouter.ai/z-ai/glm-4.5-air:free
```

Для работы с сервисом необходимо создать **API-key**. После регистрации его можно создать в правом верхнем углу (профиле) в разделе **Keys**.

⚠️ Не указывайте ключи напрямую в коде, так как это способ идентификации в приложении.
Для этого можно:

* использовать блок **переменных окружения в Colab**
* или удалить ключ из кода после выполнения.

# Задание 1. Парсинг веб-страницы

Получить **чистый текст статьи** без HTML-разметки, меню и рекламы.

### Шаги

1. Установите необходимые зависимости:

```
requests
beautifulsoup4
```

2. Напишите парсер самостоятельно **или используйте готовый из дополнительных материалов**.

Парсер должен:

* скачать HTML по ссылке на статью
* извлечь **чистый текст**:

  * без HTML-тегов
  * без навигации
  * без рекламы
  * без прочего шума

Ссылка на статью:

```
https://habr.com/ru/articles/923096/
```

3. Запустите парсер для статьи по ссылке из дополнительных материалов.

4. Убедитесь, что результат — **читаемый связный текст без HTML-мусора**.

5. Сохраните итоговый текст в файл:

```
article.txt
```

# Задание 2. Структурированная суммаризация

Получить от модели **резюме статьи строго по заданной схеме**.

### Шаги

1. Прочитайте содержимое `article.txt` в переменную.

2. Составьте запрос к модели, включающий:

* **системный промпт**
* **схему суммаризации**

Можно написать самостоятельно или воспользоваться готовыми материалами.

3. Настройте клиент **OpenRouter** (совместим с OpenAI SDK) и отправьте запрос к модели.
   Пример можно найти в приложенных материалах.

4. Выведите результат в ячейке и проверьте:

* ответ **строго следует структуре схемы**
* отражает **только содержание статьи**

5. Сформулируйте **вывод о качестве суммаризации**:

* самостоятельно
* или с помощью другой LLM (в качестве судьи)

Во втором случае **код и вывод также должны быть представлены** в работе.

# Задание 3. Абстрактная суммаризация

Получить **свободный пересказ статьи своими словами**, без привязки к схеме.

### Шаги

1. Используйте тот же текст статьи из части 2.

2. Составьте новый запрос к модели с промптом на **абстрактную суммаризацию**.

Требования:

* связный текст
* **без списков**
* **без заголовков**

3. Отправьте запрос через тот же клиент **OpenRouter** и выведите результат в ячейке.

4. Сравните два результата:

* **структурированную**
* **абстрактную** суммаризацию

И прокомментируйте различия.

# Критерии оценки практических заданий

### Оценка "Зачтено"

* задания выполнены **в полном объёме**
* полностью правильно или с **незначительными ошибками** (не более двух)
* студент демонстрирует знание **теоретического материала**
* **основные термины использованы корректно**

### Оценка "Не зачтено"

* задания выполнены **не полностью**
* есть **существенные ошибки**
* или более **трёх несущественных ошибок**
* ответ носит **поверхностный характер**
* присутствуют **неточности в терминологии**
