````md
# Практическая работа

В работе представлены **3 варианта задания**, из которых необходимо выбрать **один**.  
Каждый вариант включает:
- реализацию модели или сервиса;
- тестирование;
- подготовку краткой документации.

Допускается:
- предложить собственный вариант (с API);
- использовать работу как основу итогового проекта (с согласованием преподавателя).

# Вариант 1 — Сервис компьютерного зрения

## Задача
Разработать REST-сервис для задачи CV (классификация или детекция).

## Задание 1 — Постановка задачи и данные
- Определить задачу (например):
  - классификация товаров;
  - детекция дефектов;
  - классификация блюд.
- Описать:
  - входные данные (форматы, размеры);
  - целевые классы.
- Подготовить датасет.

## Задание 2 — Модель CV
- Выбрать модель:
  - pretrained CNN (ResNet, EfficientNet, MobileNet);
  - YOLO;
  - или модель с нуля.
- Реализовать:
  - препроцессинг (resize, нормализация);
  - инференс.
- Сохранить модель (.pt, .pth, .pkl).

## Задание 3 — Backend
FastAPI сервис:
- `GET /health` → `"ok"`
- `POST /predict`:
```json
{
  "class": "string",
  "probability": 0.95,
  "all_classes": {
    "class_1": 0.95,
    "class_2": 0.03
  }
}
````

Требования:

* валидация входа;
* обработка ошибок.

## Задание 4 — Запуск

```bash
pip install -r requirements.txt
uvicorn main:app --reload
```

Адреса:

* [http://127.0.0.1:8000](http://127.0.0.1:8000)
* [http://127.0.0.1:8000/docs](http://127.0.0.1:8000/docs)

## Задание 5 — Тестирование

* примеры запросов;
* README:

  * описание;
  * архитектура;
  * запуск;
  * примеры.

# Вариант 2 — LLM-сервис с RAG

## Задача

Backend-сервис с RAG для QA по документам.

## Задание 1 — Данные

* Выбрать область:

  * регламенты;
  * документация;
  * учебные материалы.
* Подготовить 5–20 документов.
* Сформулировать вопросы.

## Задание 2 — RAG-пайплайн

* Разбивка на чанки;
* Эмбеддинги (sentence-transformers);
* Хранилище (FAISS, Chroma);
* Поиск top-k;
* Формирование промпта;
* Подключение LLM.

## Задание 3 — Backend

FastAPI:

* `GET /health`
* `POST /ask`:

```json
{
  "question": "..."
}
```

Ответ:

```json
{
  "answer": "...",
  "sources": [
    {
      "doc_id": "policy_01",
      "score": 0.87,
      "snippet": "..."
    }
  ]
}
```

Требования:

* обработка ошибок;
* лимиты;
* логирование.

* Выбрать:

  * скоринг (вероятность / время);

  * датасет.

* train/valid/test split;
* препроцессинг;

* гиперпараметры;

## Задание 3 — Backend

* `GET /health`
* `POST /predict` или `/recommend`

```json
{
  "user_id": 123,

}
```

Требования:

* валидация;
